# 自我进化机制说明

## 🧬 Priority Coach 是如何"自我进化"的？

**核心思想：** Priority Coach 不仅是一个工具，更是一个会学习的智能助手。随着你的使用，它会：

1. **收集数据** - 记录你的决策和结果
2. **学习模式** - 识别你的个人偏好和工作模式
3. **优化算法** - 自动调整参数以更好地服务你
4. **展示成长** - 向你展示它的"学习进度"

---

## 📊 自我进化的三个层次

### 层次1: 数据收集（持续进行）

**何时收集：** 每次使用Skill时

**收集内容：**
- 你接受的推荐vs修改的推荐
- 任务完成的实际时间vs估算时间
- 你主动发起的任务vs被分配的任务
- 一天中不同时段的完成情况
- 任务完成的成功度

**存储位置：** `~/.claude/priority-coach/evolution/metrics.jsonl`

**隐私保护：** 所有数据仅存储在本地，不上传云端

---

### 层次2: 模式学习（需要7-30天数据）

#### 学习1: 你的决策风格

**Maximizer（最大化者）**
- 特征：经常询问"还有什么选择？"
- 偏好：看到多个选项后再决策
- 策略：我会提供多个排序选项

**Satisficer（满足者）**
- 特征：快速接受推荐
- 偏好：快速决策，减少思考
- 策略：我直接给出最佳3个任务

#### 学习2: 你的精力模式

**自动识别：**
- 什么时候精力最高？（如：9-11点）
- 什么时候效率最低？（如：15-17点）
- 一周中哪天效率最高/最低？

**应用：**
- 重要任务安排在精力高峰期
- 琐碎任务安排在精力低谷期

#### 学习3: 你的任务偏好

**学习内容：**
- 哪类任务你最喜欢？（如：编程 > 设计）
- 哪类任务你最容易推迟？（如：文档写作）
- 什么情况下你会主动完成任务？

**应用：**
- 预测你会推迟的任务，提前预警
- 高兴趣任务可以用来激励低兴趣任务

#### 学习4: 你的灵活性需求

**高灵活性（>0.7）:**
- 特征：经常改变计划
- 策略：增加20%留白时间

**低灵活性（<0.3）:**
- 特征：严格按计划执行
- 策略：生成详细时间表

---

### 层次3: 算法优化（需要30+天数据）

#### 优化1: 价值评分公式调整

**当前公式：**
```
Value = (goal_weight × relevance) × (urgency × 0.3 + importance × 0.7)
```

**可能的优化方向：**
- 如果你经常完成紧急但不重要的任务 → 提高urgency权重
- 如果你更看重长期价值 → 提高importance权重
- 添加难度、兴趣度等新维度

**触发条件：** 收集到30天数据后自动运行

#### 优化2: 时间估算校准

**学习：** 你通常低估还是高估时间？

**校准策略：**
- 低估 → 估算 × 1.2
- 高估 → 估算 × 0.9
- 精确 → 保持原样

**效果：** 时间估算越来越准确

#### 优化3: A/B测试新算法

**实验流程：**
1. 创建新算法版本
2. A/B测试7天（一半用户用新算法，一半用旧算法）
3. 对比性能指标（完成率、满意度等）
4. 如果新算法提升>5%，自动切换

**持续进行：** 每2-4周测试一次新想法

## 🧠 学习机制详解

### 机制1: 决策风格识别

**检测逻辑：**
```python
# 伪代码
info_request_count = count("用户询问更多信息")
total_interactions = count("所有交互")

if info_request_count / total_interactions > 0.3:
    user_type = "Maximizer"
else:
    user_type = "Satisficer"
```

**Maximizer示例对话：**
```
你: priority-coach 帮我规划今天

我: 📋 今日建议（3个选项）：

【选项A：价值优先】
1. 核心算法 [4番茄钟] - 价值5分
2. 测试修复 [2番茄钟] - 价值5分
3. 文档编写 [1番茄钟] - 价值3分

【选项B：效率优先】
1. 测试修复 [2番茄钟] - 价值5分
2. 核心算法 [4番茄钟] - 价值5分
3. 文档编写 [1番茄钟] - 价值3分

【选项C：难度递增】
1. 文档编写 [1番茄钟] - 价值3分
2. 测试修复 [2番茄钟] - 价值5分
3. 核心算法 [4番茄钟] - 价值5分

你更喜欢哪个方案？
```

**Satisficer示例对话：**
```
你: priority-coach 帮我规划今天

我: 🎯 今日聚焦这3件事：

1. 核心算法 [4番茄钟] - 价值5分
2. 测试修复 [2番茄钟] - 价值5分
3. 跑步5公里 [1.5番茄钟]

⏰ 时间表已生成，开始吧！
```

### 机制2: 精力模式学习

**数据收集：**
```

2025-03-03 14:00-15:00  完成任务C（成功率85%） → 中能量时段
...
```

**学习曲线（示例）：**
```
精力水平
  ^

**应用：**

```

### 机制3: A/B测试框架

**测试示例：**

**问题：** 应该给用户更多选项还是更少选项？

**实验设计：**
- **Variant A（对照组）**: 当前快速推荐（直接给3个任务）
- **Variant B（实验组）**: 提供多个选项让用户选择

**运行时间：** 7天

**数据收集：**
- 用户满意度（1-5分）
- 完成率
- 决策时间

**结果分析：**
```
实验结果：

用户满意度:
- Variant A: 4.2/5.0
- Variant B: 4.6/5.0 (+9.5%)

完成率:
- Variant A: 75%
- Variant B: 78% (+4%)

决策时间:
- Variant A: 30秒
- Variant B: 90秒 (3倍)

结论: Variant B 用户更满意，但决策时间增加
推荐: 对于Satisficer用户使用A，对于Maximizer用户使用B
```

**进化：**
- 如果Variant B显著提升 → 为所有用户启用选项模式
- 如果Variant A效果更好 → 保持当前策略
