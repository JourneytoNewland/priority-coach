## 示例1: 首次使用 - 初始化目标

**场景：** 第一次使用priority-coach

```

学会空翻
每月跑100公里

我: 太好了！让我记录下你的目标：

**🎯 你的3个月目标：**
1. 大数据问数智能体及应用开发 - 3月底交付上线

**这些事情对你的长期意义是什么？**

我: 完美！让我记录：

## 示例4: 下午调整 - 临时插入任务

**场景：** 下午突然有紧急任务插入

```
你: priority-coach 计划有变，突然有个紧急bug需要现在修

我: ⚠️ 临时任务插入

让我评估一下：

**新任务：** 紧急bug修复
- 价值分数：5分（紧急且重要）
- 预估时间：2番茄钟

**当前今日计划：**
14:00-15:30 大数据智能体开发 [3番茄钟]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💡 调整建议：

**方案A：立即处理**（推荐）
- 14:00-15:00 紧急bug修复 [2番茄钟]
- 15:00-15:10 休息
- 15:10-16:30 继续原计划（压缩番茄钟数）

**方案B：使用留白时间**
- 14:00-16:00 继续原计划
- 17:10-18:00 使用留白时间修复bug

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 影响：

原计划：9.5番茄钟
方案A：9番茄钟（减少0.5个）
方案B：9.5番茄钟（不变）

🎯 推荐：方案A
- 紧急bug优先处理
- 下午精力更适合排查问题
- 原计划可适当延后

你选择哪个方案？（A/B/或者告诉我其他想法）
```

## 示例7: 目标检查 - 验证对齐

**场景：** 想要验证今天的任务是否与长期目标对齐

```
你: priority-coach 目标检查

我: 🎯 目标对齐检查

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

基于你的长期目标：
1. 大数据问数智能体（3月31日上线）
2. 每月跑100公里
3. 学会空翻

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**今日任务分析:**

✅ 核心算法开发
→ 直接推进目标1 (价值5分)
→ 正确！这是最重要的事

✅ 技术文档编写
→ 支持目标1 (价值4分)
→ 重要但不紧急，可以延后

⚠️  邮件处理1.5小时
→ 维持性工作 (价值2分)
→ 可优化

❌ 反复检查通知
→ 逃避行为 (价值1分)
→ 建议删除

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**💡 优化建议:**

1. **邮件改为每日3次集中处理**
   - 早/中/晚各15分钟
   - 节省时间给高价值任务

2. **关闭非紧急通知**
   - 减少干扰
   - 将节省的时间用于核心算法开发

3. **技术文档可以拆解**
